    THE CHALLENGE  About This Competition  Online review platforms are among the richest sources of human behaviour data on the web. Every rating, every review, every browsing session is a signal — a window into preference, context, and decision-making. Yet most AI systems still treat users as static profiles rather than dynamic, context-sensitive agents.  This challenge is our investment in changing that. We are looking for the sharpest minds working on LLM based user modeling and intelligent recommendation — and we want to give them the stage to do their best work.  Top performers will be invited to a talent conversation with Bluechip Tech’s data and AI practice leads.       COMPETITION STRUCTURE  Two Task. One Ambition.  TASK A  User Modeling  Build an agent that understands users deeply enough to simulate their reviews — capturing tone, rating behaviour, and contextual nuance.  ›  Simulate star ratings and written reviews for unseen items  ›  Leverage user history, item metadata, and contextual signals  ›  Evaluated on review quality, rating accuracy, and behavioural fidelity  TASK B  Recommendation  Build an agent that delivers personalised recommendations — going beyond collaborative filtering to contextual, conversational retrieval.  ›  Rank and recommend items tailored to individual user context  ›  Handle cold-start, cross-domain, and multiturn scenarios  ›  Design agentic workflows that reason before recommending    Where you can get Dataset: Yelp, Amazon Reviews, and Goodreads — three large-scale, real world platforms with rich behavioral and textual signals.

DELIVERABLES  Three Submissions. All Required.   Task 1: You are required to submit a containerized application with either a web application or an API      endpoint. The system should take user persona and product details as input, and generate reviews   and ratings as output.  Task 2: You are required to submit a containerized application with either a web application or an API endpoint. The system should take user persona as input and produce personalized recommendations as output. (Examples of product can include Movies, food, drinks etc)      Solution Paper (4–8 Pages)  02  A write-up covering your approach, architecture decisions, experiments run, ablation studies, and what could be done with more time. This is the primary talent signal and what judges read first. Originality and clarity of reasoning are rewarded above raw performance.       Code Repository  03    A clean, documented, reproducible codebase submitted via GitHub or equivalent. Judges will attempt to run it. Bonus credit is awarded for a clear README, modular design, and well commented agentic workflow logic.
